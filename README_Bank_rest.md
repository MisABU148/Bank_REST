# Система управления банковскими картами

## Описание проекта

Backend-приложение для управления банковскими картами пользователей.
Система позволяет администраторам управлять пользователями и картами, а пользователям — просматривать свои карты, получать информацию о балансе и выполнять переводы между собственными картами.

Приложение разработано на **Java + Spring Boot** с использованием **Spring Security и JWT-аутентификации**.

---

# Функциональность системы

## Аутентификация и авторизация

В системе реализована аутентификация и авторизация пользователей на основе **JWT-токенов**.

Особенности реализации:

* используется **Spring Security**
* применяется **JWT access token и refresh token**
* токены выдаются после успешной авторизации
* пароли пользователей **кодируются с использованием BCrypt**
* доступ к API ограничен **ролевой моделью**

Поддерживаемые роли:

* **ADMIN**
* **USER**

---

# Возможности системы

## Администратор

Администратор имеет полный доступ к системе и может:

* создавать пользователей
* обновлять данные пользователей
* удалять пользователей
* просматривать список пользователей

Также администратор может управлять банковскими картами:

* создавать карты
* просматривать карту по id
* получать список всех карт (с пагинацией)
* удалять карты

---

## Пользователь

Пользователь имеет доступ только к собственным картам и может:

* получать список своих карт (с постраничной выдачей)
* просматривать баланс карты
* запрашивать блокировку карты
* выполнять переводы между своими картами

---

# Работа с банковскими картами

Каждая карта содержит следующие атрибуты:

* номер карты
* владелец карты
* срок действия
* статус карты
* баланс

Номер карты хранится в системе в защищённом виде и отображается с маской:

```
**** **** **** 1234
```

Статусы карт:

* ACTIVE
* BLOCKED
* EXPIRED

# API

В системе реализованы REST API endpoints для:

* управления пользователями (CRUD)
* управления банковскими картами (CRUD)
* получения баланса карты
* блокировки карты
* перевода средств между своими картами
* получения списка карт пользователя
* постраничного получения данных

Все входные данные проходят **валидацию**.

Ошибки обрабатываются и возвращаются в виде корректных HTTP-ответов.

# Docker

Для удобного запуска проекта используется **Docker Compose**.

Docker поднимает:

* приложение Spring Boot
* базу данных PostgreSQL

Запуск проекта:
- Создайте файл .env в корне проекта:

```.env
# PostgreSQL
POSTGRES_DB=bankcards
POSTGRES_USER=bankuser
POSTGRES_PASSWORD=bankpass

# Spring приложение
SPRING_DATASOURCE_PASSWORD=bankpass
```
- Сборка и запуск
```bash
mvn clean package
docker-compose up --build
```

После запуска API будет доступно по адресу:

```
http://localhost:8080
```
